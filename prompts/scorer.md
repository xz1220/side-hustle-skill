# 副业评分标准

## 6 维度评分体系

每个维度 1-5 分，满分 30 分。打分必须结合用户画像——同一个副业方向对不同用户分数不同。

### 维度 1: 技能匹配度

评估用户现有技能和该方向所需技能的重合度。

| 分数 | 标准 |
|------|------|
| 5 | 用户已有的技能可以直接变现，几乎不需要额外学习 |
| 4 | 核心技能匹配，只需补充少量周边知识（< 1 周） |
| 3 | 有相关基础，需要 1-4 周学习新技能 |
| 2 | 有间接关联，需要 1-3 个月系统学习 |
| 1 | 几乎从零开始，需要 3 个月以上学习 |

### 维度 2: 时间适配度

评估用户可用时间是否能支撑该副业。

| 分数 | 标准 |
|------|------|
| 5 | 碎片时间就能做，完全适配用户时间安排 |
| 4 | 需要固定时间段但总量在用户预算内 |
| 3 | 刚好够用，但几乎没有余量 |
| 2 | 需要时间超出用户预算 20-50% |
| 1 | 需要时间远超用户预算，不可行 |

### 维度 3: 启动成本

评估启动所需资金与用户预算的匹配度。

| 分数 | 标准 |
|------|------|
| 5 | 零成本启动，只需要时间和已有设备 |
| 4 | 需要少量投入（< 200 元），在用户预算内 |
| 3 | 需要中等投入（200-1000 元），接近用户预算上限 |
| 2 | 需要较多投入（1000-5000 元），超出用户预算 |
| 1 | 需要大额投入（> 5000 元），远超用户预算 |

### 维度 4: 收入天花板（基于三元组）

评估该方向的收入潜力。**基于 `knowledge/categories.md` 的三元组模型**（P50 首桶金 / 12mo 存活率 / P95），不使用单一"月收入区间"。

**打分基准：** 主锚 = `P50 首桶金` + `12mo 存活率`。**P95（头部天花板）只在匹配理由叙事里简要提及，不作打分依据。**

**数据处理规则：**
1. **案例库优先**：对应方向在 `knowledge/cases/` 有 P50 案例时（或 `categories.md` 有"参考案例"列），以案例数据为主锚
2. **置信度校验**：`categories.md` 该行置信度为 `L` 或 `last_verified > 90 天` → **必须 WebSearch 补数据后再打分**，不得仅凭本行给 4-5 分
3. **源可信度校验**（见 `source_trust.md`）：搜索数据必须至少 1 个 Tier 1 或 2 个独立 Tier 2 支撑
4. **冲突处理**：多个来源差距 > 3 倍时，取较低值（"保守而非乐观"原则）
5. **头部案例降权**：只找到 P99 案例（如 Marc Lou、Plausible）而无 P50 数据时，不得用 P99 反推出"4 分"；应降至 3 分并在理由里说明"仅有头部案例，P50 数据不足"

| 分数 | 标准（P50 + 存活率综合）|
|------|------------------------|
| 5 | P50 ≥ ¥5000 或 $1000 MRR，且 12mo 存活率 ≥ 30% |
| 4 | P50 ¥2000-5000 或 $500-1000 MRR，且 12mo 存活率 ≥ 20% |
| 3 | P50 ¥1000-2000 或 $200-500 MRR；或存活率 < 20% 但天花板高 |
| 2 | P50 ¥500-1000；或存活率 < 10% |
| 1 | P50 < ¥500；或无直接变现路径（如纯 reputation/开源）|

**引用来源（必填，否则不得给 4/5 分）：** 在评分理由里注明三元组出处 + 置信度 + 源等级，例如：

```
收入天花板 3 — P50 $100-500 MRR，12mo 存活 20% [categories.md 独立开发行, 置信 H;
案例 resume_tool_35yo.md 验证 ¥3000/月 是 P50 而非下界]
```

**特殊方向（无直接变现）：** 如 Claude Skill / AI 生态开源这类 MIT 免费方向，直接给 **1 分**（符合"无直接变现路径"）；但在匹配理由里说明其作为 reputation 杠杆、间接反哺其他方向的战略价值。

### 维度 5: 冷启动难度

评估从零到第一笔收入的难度和时间。

| 分数 | 标准 |
|------|------|
| 5 | 一周内可以出第一单/第一笔收入 |
| 4 | 2-4 周内可以出第一笔收入 |
| 3 | 1-2 个月内可以出第一笔收入 |
| 2 | 3-6 个月才能看到收入 |
| 1 | 6 个月以上才可能有收入，前期全是投入 |

### 维度 6: 可持续性

评估该副业的长期价值。

| 分数 | 标准 |
|------|------|
| 5 | 可复利增长（内容/品牌/客户积累越做越值钱） |
| 4 | 收入稳定可持续，有一定积累效应 |
| 3 | 收入相对稳定，但没有积累效应（按件计费） |
| 2 | 收入不稳定，容易被替代（价格竞争激烈） |
| 1 | 短期窗口期，很快会被市场/技术淘汰 |

## 输出格式

按总分降序排列，每个方向使用统一 Markdown 格式（不使用 ASCII 盒子——在中英文混排时对齐不稳）：

```markdown
## #{排名} {方向名称}  {总分}/30 分

评分明细：
- 技能匹配 {分} — {一句话理由}
- 时间适配 {分} — {一句话理由}
- 启动成本 {分} — {一句话理由}
- 收入天花板 {分} — {一句话理由}{若 4/5 分须注数据来源}
- 冷启动 {分} — {一句话理由}
- 可持续性 {分} — {一句话理由}

匹配理由：{一句话}
风险提示：{一句话}
冷启动第一步：{具体行动，含平台 + 动作}
```

## 评分原则

1. **基于数据打分** — 每个分数都要有搜索数据或知识库支撑
2. **个性化打分** — 同一方向对不同用户分数不同（技能匹配、时间适配因人而异）
3. **保守而非乐观** — 宁可低估不要高估，用户信任比好看更重要
4. **说明理由** — 每个维度的分数都要有一句话解释为什么
