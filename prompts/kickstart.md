# 冷启动方案生成模板

## 目标

为用户选定的副业方向生成具体的、可执行的冷启动 checklist。不说废话，每一条都是具体动作。

## 生成前准备

1. 结合用户画像中的技能、时间、预算
2. 用 WebSearch 搜索该方向的冷启动经验：
   - `"{方向}" 新手 冷启动 经验`
   - `"{方向}" 第一单 怎么来的`
   - `"{方向}" 从零开始 教程`
3. 提取可操作的步骤，过滤掉空话

## 输出模板

```markdown
# {方向} 冷启动方案

> 基于你的画像定制：{技能} + {时间}/周 + {预算}预算

## 准备阶段（第 0 周）

- [ ] {具体动作 1}（预计 X 小时）
- [ ] {具体动作 2}（预计 X 小时）

## 第一周：{本周目标}

- [ ] {具体动作}
- [ ] {具体动作}
- [ ] {具体动作}

## 第二周：{本周目标}

- [ ] {具体动作}
- [ ] {具体动作}

## 第三~四周：{阶段目标}

- [ ] {具体动作}
- [ ] {具体动作}

## 第一个月检查点

- [ ] 完成了：{milestone}
- [ ] 如果没达成，可能是因为：{常见原因}
- [ ] 调整方向：{建议}

## 需要的工具/平台

| 工具 | 用途 | 费用 |
|------|------|------|
| {tool} | {purpose} | {cost} |

## 预期时间线

- 第一笔收入：约 {X} 周
- 月入 1000：约 {X} 个月
- 稳定收入：约 {X} 个月

## 常见坑

1. {坑 1}：{怎么避}
2. {坑 2}：{怎么避}
3. {坑 3}：{怎么避}
```

## 生成原则

1. **每个 action 都是具体动作** — "注册小红书账号并发第一篇帖子"，不是"开始做内容"
2. **标注时间成本** — 让用户知道每个步骤大概花多久
3. **符合用户约束** — 推荐的工具要免费或在预算内，时间安排要在用户可用时间内
4. **给出检查点** — 每个阶段告诉用户"做到什么程度算合格"
5. **预设失败路径** — 告诉用户如果某步卡住了怎么办
