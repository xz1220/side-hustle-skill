# 副业方向搜索与分析方法论

## 目标

基于用户画像，系统化搜索和筛选适合的副业方向。

## 搜索流程

### Phase 1: 技能映射

将用户的每项技能映射到可能的变现方向：

```
用户技能 → 直接变现场景（有这个技能就能赚钱）
         → 间接变现场景（技能 + 少量学习就能做）
         → 跨界组合场景（多个技能组合产生独特价值）
```

参考 `knowledge/categories.md` 中的品类库做映射。

### Phase 2: 约束过滤

用硬约束过滤掉不可行的方向：

- **时间约束**：需要 > 用户可用时间的，排除
- **资金约束**：启动成本 > 用户预算的，排除
- **排除项**：命中用户排除项的，排除
- **技能差距过大**：需要 > 3 个月学习才能入门的，标记但不排除

### Phase 2.5: 案例库查询（优先于 WebSearch）

在做 WebSearch 之前，先查 `knowledge/cases/` 是否已有对应方向的真实案例。

**目的：**
- 避开 WebSearch 的 SEO 污染
- 使用经过人工核验的真实数据作为锚点
- 同一方向最好同时找到 P50（中位数）和 P95（头部）案例，**避免被头部案例误导**

**查询方式：**
1. 打开 `knowledge/cases/README.md` 的案例索引表
2. 按 `category` 字段找到与当前方向匹配的案例（优先 P50，再补 P95、failed）
3. 读相关 case 文件，提取 "评分启示" 段落
4. 如果 `categories.md` 对应行有"参考案例"列，直接按链接跳转

**组合使用：**
- 案例库有 P50 案例 → 以案例数据为收入维度主锚点；WebSearch 仅补规则/时效
- 案例库无 P50 案例 → WebSearch 补数据，并考虑回填到 `categories.md` 和 `cases/`

### Phase 3: 市场验证（WebSearch）

对通过过滤的每个方向，搜索以下信息：

**搜索关键词模板：**
- `"{方向}" 副业 收入` — 了解真实收入水平
- `"{方向}" 新手 入门 怎么开始` — 了解入门门槛
- `"{方向}" 2025 2026 还能做吗` — 了解市场饱和度
- `"{方向}" 副业 经验分享` — 找真实案例

**信息提取：**
- 收入范围（新手 / 中等 / 高手）
- 入门所需时间
- 主要平台和渠道
- 常见坑和风险
- 成功案例的共同特征

**可信度与来源处理：完整规则见 `knowledge/source_trust.md`**（Tier 分层、域名黑白名单、红旗短语字典、交叉验证规则）。

**执行摘要（必读）：**
- **Tier 1**（平台官方 / 官方报告 / IndieHackers Open Startups / 政策文件）→ 单源可采信
- **Tier 2**（具名专栏 / 深度复盘 / V2EX / 少数派 / 知乎实名专栏 / Medium 实名博客）→ 需 2+ 独立源交叉
- **Tier 3**（`*-cc.net` / `kezhitan.com` 等 SEO 站群、CSDN 广告贴、无作者+导流贴、"月入过万"文）→ **一律忽略**，不得作为评分依据
- 单篇命中红旗短语（"零基础月入过万""日赚 30-50""私信领取""加群领资料"等）→ 自动降级一 tier
- 头部案例单独放大而不讲中位数 → 不得作为收入锚点
- 无 Tier 1/2 来源时：宁可在评分理由里标"数据不足"，也不采信 Tier 3

### Phase 4: 候选列表

综合以上信息，筛选出 5-10 个候选方向，准备进入评分阶段。

每个候选方向需要包含：
- 方向名称（具体，不笼统）
- 为什么匹配这个用户（映射逻辑）
- 搜索到的关键数据点
- 初步判断的风险

## 搜索原则

1. **具体 > 笼统**：不推荐"做自媒体"，推荐"在小红书做 XX 领域图文"
2. **已验证 > 理论上**：优先推有真实案例的方向
3. **低门槛 > 高门槛**：用户是新手，优先推容易起步的
4. **多元化**：推荐结果覆盖不同类型（内容创作、技能服务、数字产品、线下服务等）
