# 副业方向搜索与分析方法论

## 目标

基于用户画像，系统化搜索和筛选适合的副业方向。

## 搜索流程

### Phase 1: 技能映射

将用户的每项技能映射到可能的变现方向：

```
用户技能 → 直接变现场景（有这个技能就能赚钱）
         → 间接变现场景（技能 + 少量学习就能做）
         → 跨界组合场景（多个技能组合产生独特价值）
```

参考 `knowledge/categories.md` 中的品类库做映射。

### Phase 2: 约束过滤

用硬约束过滤掉不可行的方向：

- **时间约束**：需要 > 用户可用时间的，排除
- **资金约束**：启动成本 > 用户预算的，排除
- **排除项**：命中用户排除项的，排除
- **技能差距过大**：需要 > 3 个月学习才能入门的，标记但不排除

### Phase 3: 市场验证（WebSearch）

对通过过滤的每个方向，搜索以下信息：

**搜索关键词模板：**
- `"{方向}" 副业 收入` — 了解真实收入水平
- `"{方向}" 新手 入门 怎么开始` — 了解入门门槛
- `"{方向}" 2025 2026 还能做吗` — 了解市场饱和度
- `"{方向}" 副业 经验分享` — 找真实案例

**信息提取：**
- 收入范围（新手 / 中等 / 高手）
- 入门所需时间
- 主要平台和渠道
- 常见坑和风险
- 成功案例的共同特征

**可信度判断：**
- "月入十万"、"躺赚"、"零基础月入过万" → 降权，可能是引流内容
- 有具体数据、时间线、过程描述的 → 可信度高
- 多个独立来源交叉验证的数据 → 可采信

**来源白名单（优先采信）：**
- 知乎专栏（zhuanlan.zhihu.com）、即刻、少数派、电鸭社区、V2EX
- 具名个人博客（含作者名+发布日期）、Medium
- 产品方官方博客（如 Figma / Gumroad / 小红书千帆 官方）

**来源黑名单（直接忽略，不采信其数据）：**
- 域名属于课程聚合/网赚站（如 `*-cc.net`、`kezhitan.com`、`quan520.com` 等）
- 标题含"课程合集打包""网赚大全""副业开挂指南""日赚 30-50 元"
- 整站内容以引流付费课程为主的营销号

**强制交叉验证规则：**
- 白名单内 1 个来源 → 可直接采信
- **非白名单来源 → 至少需要 2 个独立白名单来源交叉验证，才能作为评分依据**
- 评分维度涉及收入/市场规则/平台门槛时，优先引用官方来源或近 90 天内的白名单内容

### Phase 4: 候选列表

综合以上信息，筛选出 5-10 个候选方向，准备进入评分阶段。

每个候选方向需要包含：
- 方向名称（具体，不笼统）
- 为什么匹配这个用户（映射逻辑）
- 搜索到的关键数据点
- 初步判断的风险

## 搜索原则

1. **具体 > 笼统**：不推荐"做自媒体"，推荐"在小红书做 XX 领域图文"
2. **已验证 > 理论上**：优先推有真实案例的方向
3. **低门槛 > 高门槛**：用户是新手，优先推容易起步的
4. **多元化**：推荐结果覆盖不同类型（内容创作、技能服务、数字产品、线下服务等）
