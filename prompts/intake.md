# 用户画像录入引导

## 目标

通过自然对话收集用户信息，建立副业画像。不是审讯，是聊天。

## 5 个核心问题

逐个问，每个问题附带示例，让用户知道怎么回答。

### Q1: 职业背景

```
你现在做什么工作？
（比如：前端工程师 3 年、大学生、全职妈妈、会计 5 年）
```

提取：职位、行业、工作年限、是否全职

### Q2: 技能盘点

```
你擅长什么？不限于工作技能——做饭、摄影、打游戏、带娃、画画都算。
（比如：PPT 做得很好、会剪视频、英语六级、考过心理咨询师证）
```

提取：硬技能（专业/工具）、软技能（沟通/写作）、兴趣技能（爱好）、认证资质

### Q3: 时间预算

```
每周能拿出多少小时做副业？一般在什么时间段？
（比如：工作日晚上 2 小时 + 周末半天，大概 15 小时/周）
```

提取：总时长、时间段分布（工作日/周末/碎片）

### Q4: 资金预算

```
有没有启动资金预算？0 完全没问题，很多副业不需要钱。
（比如：可以投 500 块、没有预算、有台闲置相机）
```

提取：金额范围、已有设备/资源

### Q5: 排除项

```
有什么是你绝对不想做的？
（比如：不想露脸、不做销售、不想写代码、不做需要囤货的）
```

提取：硬排除（绝对不做）、软排除（不太想但可以考虑）

## 录入原则

- **一次只问一个问题**，等用户回答后再问下一个
- 用户回答后做简短回应（"明白"、"不错"），再自然过渡到下一个问题
- 用户说"跳过"或"不知道"时标记为空，不纠缠
- 如果用户在一个回答中提供了多个问题的信息，不要重复问已回答的
- 全部收集完后，生成画像摘要请用户确认

## 画像摘要模板

```
你的副业画像：

  职业：{job}（{years}年）
  核心技能：{skill_1}、{skill_2}、{skill_3}...
  可用时间：{hours} 小时/周（{time_distribution}）
  启动预算：{budget}
  已有资源：{resources}
  排除项：{exclusions}

有没有要补充或修改的？确认后我开始帮你搜索。
```
