# 信息源可信度规则

> 副业内容的 SEO 污染极重。本文件是调用 WebSearch 前后必须遵守的可信度处理规则。
> 被 `prompts/discovery.md` 引用。任何推荐报告里的收入/规则数据都必须符合本文件的标准。

---

## 一、可信度分层

### Tier 1 — 一手来源（可单源采信）

- **平台官方文档/博客**：Figma、Gumroad、小红书千帆、微信公众号官方、B 站创作中心、Upwork/Fiverr 官方、Stripe/Vercel/Supabase 等
- **官方发布的行业报告**：Stack Overflow Developer Survey、CNNIC、艾瑞咨询公开报告、平台年度创作者报告
- **公开透明收入**：IndieHackers Open Startups（作者本人公开 MRR）、Stripe Atlas 案例、作者本人的 X/Twitter 收入截图且可溯源
- **政策/规则文件**：国家网信办、市场监管总局、平台公告

**使用方式**：单个 Tier 1 来源即可作为评分依据；引用时注明发布日期。

### Tier 2 — 具名深度复盘（需 2+ 独立源交叉）

- 知乎专栏（zhuanlan.zhihu.com，作者实名 + 有历史专业输出）
- 即刻深度帖（有时间线和具体数据）
- 少数派 Matrix 文章
- 个人博客（有作者名、发布日期、可溯源历史）
- V2EX 创意区、电鸭社区、独立开发者社群的长帖
- 掘金、SegmentFault 技术复盘（作者实名）
- Medium/Dev.to 具名技术博客
- Product Hunt 发布后的作者复盘

**使用方式**：至少 2 个独立 Tier 2 来源数据一致，才能采信；单一 Tier 2 来源只作为辅助参考。

### Tier 3 — 不采信（识别后直接忽略）

识别特征（命中任一即归 Tier 3）：

- 域名黑名单：`*-cc.net`、`kezhitan.com`、`quan520.com`、CSDN 广告落地页、自动聚合类 `gitcode.csdn.net`、`cloud.tencent.com/developer` 的广告贴
- 标题红旗："课程合集打包""网赚大全""副业开挂指南""日赚 30-50 元""闭眼做 xx 月入过万""XX 偏门项目"
- 内容红旗：
  - 无作者名、无发布时间
  - 正文结尾有"加微信""进群""私信领取"
  - 通篇罗列方向、每个方向都不超过 200 字、无任何数据
  - "以上内容整理自网络"式声明
  - AI 批量生成痕迹（排比句堆砌、同义词轮换、数字凑整）
- 平台内卖课/引流贴：标题党 + 结尾导流私域

**使用方式**：见到即忽略，不得作为评分依据；**即使只有 Tier 3 来源也宁可给"数据不足"而不采信**。

## 二、单篇文章的红旗短语字典

搜索结果命中以下短语时，**至少降一级可信度**：

| 短语类型 | 典型例子 | 处理 |
|---------|---------|------|
| 收入夸大 | "月入过万""日入千元""躺赚""轻松变现""人人能做" | 降一级；若无具体数据，归 Tier 3 |
| 零基础话术 | "零基础月入过万""小白逆袭""无需经验" | 降一级 |
| 时效伪装 | 正文出现"2023/2024"旧数据但标题写"2026"；或全文无具体日期 | 降一级 |
| 导流话术 | "免费领取""私信我""加群领资料""课程限时免费" | 归 Tier 3 |
| 数据无来源 | "据统计""有数据显示""业内人士透露" 但不给原始来源 | 降一级 |
| 头部案例单独放大 | 只讲一个头部案例（月入十万）、不讲中位数和存活率 | 降一级；不得作为收入锚点 |

## 三、交叉验证规则

1. **Tier 1 单源**：可直接采信，引用时标来源 + 日期
2. **Tier 2 单源**：标记为"参考"，不单独作为评分依据
3. **Tier 2 × 2 独立源**：数据一致可采信；不一致取较保守值
4. **Tier 3**：一律忽略，不得交叉验证使用（两个 Tier 3 不等于一个 Tier 2）
5. **官方 vs 二手冲突**：官方优先；二手即使更新鲜也不压过官方

---

## 四、引用和可信度标注（写入推荐报告）

涉及收入、市场规模、平台规则的评分理由，**必须在推荐报告里注明数据来源和可信度等级**。格式：

```
收入天花板 4 — 中等水平月入 ¥3000-5000 [Tier 1: IndieHackers Open Startups, 2026-03 样本 50+]
```

或：

```
冷启动 2 — 小红书虚拟店新规粉丝≥1000 [Tier 1: 小红书千帆官方公告, 2026-02-15]
```

如无可靠来源：

```
收入天花板 2 — 保守估 ¥500-1000，[数据不足，基于品类库估算，置信度低]
```

---

## 五、当数据不足时的默认行为

1. **不凭空编数据**：没有可靠来源的方向，直接在评分理由里标注"数据不足"
2. **优先低估**：候选值在 3 分和 4 分之间时，选 3 分
3. **说明不确定度**：报告总结里注明"本轮评分中 X 个方向数据源 < 2 个 Tier 2，可能有 ±2 分漂移"
4. **允许拒绝回答**：整个方向的数据都无法找到 Tier 1/2 来源时，不推荐该方向（即便技能匹配高）

- 每次跑完 `/side-hustle` 后，如果发现新的应拉黑域名 / 新红旗短语，追加到本文件
- 搜索结果中反复出现的 Tier 1 源，可写入 `prompts/discovery.md` 的首选查询列表
- 用户报告"推荐错误"时，回看当时引用的来源等级，作为 skill 质量信号
