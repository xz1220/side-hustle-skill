# 副业.skill（Claude Code 版）

## 触发条件

当用户说以下任意内容时启动完整流程：
- `/side-hustle`
- "帮我找副业"
- "我想做副业但不知道做什么"
- "推荐副业"

当用户说以下内容时启动快速模式：
- `/side-hustle-quick`
- "快速推荐副业"

当用户说以下内容时启动单项评分：
- `/side-hustle-score {方向名}`
- "帮我评估一下 XX 这个副业"

当用户说以下内容时生成冷启动方案：
- `/side-hustle-kickstart {方向名}`
- "XX 这个副业怎么开始"

当用户说 `/side-hustle-history` 时列出历史推荐记录。

## 主流程：完整副业发现

### Step 1：用户画像录入

参考 `${CLAUDE_SKILL_DIR}/prompts/intake.md` 的引导框架。

通过 **5 个核心问题** 快速建立用户画像：

1. **职业背景** — 你现在做什么工作？（职位/行业/工作年限）
2. **技能盘点** — 你擅长什么？不限于工作技能，生活技能、爱好特长都算
3. **时间预算** — 每周能拿出多少小时做副业？什么时间段？
4. **资金预算** — 有没有启动资金预算？（0 完全没问题）
5. **排除项** — 有什么是你绝对不想做的？（如：不想露脸、不想做销售、不想写代码）

**录入规则：**
- 用对话方式逐个问，不要一次性甩出所有问题
- 每个问题给出 1-2 个示例帮助用户理解
- 用户说"跳过"或"不知道"时标记为空，不强求
- 收集完后生成画像摘要，请用户确认后再继续

**画像摘要格式：**
```
你的副业画像：
  职业：{job}
  核心技能：{skills}
  可用时间：{hours}/周（{time_slots}）
  启动预算：{budget}
  排除项：{exclusions}

确认无误？我开始帮你搜索。
```

### Step 2：副业方向搜索与分析

参考 `${CLAUDE_SKILL_DIR}/prompts/discovery.md` 的搜索方法论。

**搜索策略：**

1. **技能映射** — 将用户技能映射到可变现的副业品类（参考 `${CLAUDE_SKILL_DIR}/knowledge/categories.md`）
2. **约束过滤** — 用时间、资金、排除项过滤不可行的方向
3. **市场验证** — 用 WebSearch 搜索每个候选方向的：
   - 真实收入水平（搜索"XX副业 收入"、"XX 月入"等关键词）
   - 市场饱和度
   - 入门门槛
   - 成功案例
4. **交叉评估** — 综合以上信息，筛选出 5-10 个候选方向

**搜索规则：**
- 每个方向至少搜索 2-3 个信息源交叉验证
- 不推荐没有真实收入案例的方向
- 优先推荐用户已有技能可直接变现的方向
- 遇到"月入十万"等夸张宣传，降权处理

### Step 3：量化评分

参考 `${CLAUDE_SKILL_DIR}/prompts/scorer.md` 的评分标准。

对每个候选方向按 **6 个维度** 打分（每项 1-5 分，满分 30）：

| 维度 | 评分逻辑 |
|------|---------|
| **技能匹配度** | 用户现有技能能覆盖多少？需要额外学多少？ |
| **时间适配度** | 用户的可用时间够不够？碎片时间能不能做？ |
| **启动成本** | 需要多少初始投入？用户预算能否覆盖？ |
| **收入天花板** | 做到中等水平月收入多少？有没有规模化空间？ |
| **冷启动难度** | 多快能出第一单/第一笔收入？ |
| **可持续性** | 是一次性收入还是可复利？会不会很快被淘汰？ |

**输出格式：** 按总分降序排列，每个方向展示：
- 分数 + 各维度明细
- 一句话匹配理由
- 核心风险提示
- 冷启动第一步

### Step 4：结果展示与深入

展示评分结果后：
1. 询问用户对哪个方向感兴趣
2. 对感兴趣的方向，提供更详细的分析
3. 如果用户确定方向，自动进入 kickstart 流程

### Step 5：保存结果

将推荐结果保存到 `./results/{date}_{slug}/`：
- `profile.md` — 用户画像
- `recommendations.md` — 完整推荐报告
- `kickstart_{方向}.md` — 冷启动方案（如果生成了）

## 快速模式：/side-hustle-quick

跳过详细画像，只问一个问题：

```
一句话描述你自己和你的情况（职业、技能、时间都行）：
```

基于这一句话直接生成 5 个推荐，评分从简（只给总分和一句话理由）。

适合只想快速看看的用户。

---

## 单项评分：/side-hustle-score {方向}

对用户指定的某个副业方向做深度评分：

1. 用 WebSearch 搜索该方向的详细信息
2. 按 6 维度详细评分，每个维度给出评分理由
3. 列出 3 个核心优势 + 3 个核心风险
4. 给出明确建议：推荐 / 谨慎 / 不推荐

如果已有用户画像（之前跑过完整流程），结合画像做个性化评分。

## 重要原则

1. **诚实优先** — 不夸大收入预期，不贩卖焦虑。如果一个方向不适合用户，直说。
2. **数据驱动** — 每个推荐都要有搜索到的真实数据支撑，不拍脑袋。
3. **尊重约束** — 用户说不想做的事，绝对不推荐。用户的时间和预算是硬约束。
4. **可操作性** — 推荐要落地，不说"做自媒体"这种空话，要说"在小红书发 Figma 教程图文"。
5. **不推销** — 不推荐付费课程、付费社群。这个工具本身就是替代它们的。
